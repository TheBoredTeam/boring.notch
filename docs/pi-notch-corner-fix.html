Notch corner fix — peek & Composio tab

While the agent runs, the notch read as a squared-up box instead of a rounded notch. This page renders the real geometry — the NotchShape silhouette, the closed-state inset, and the aurora glow — so you can see exactly where the square came from and what the fix changes. before the squared look

The aurora is the peek's .background(), but ContentView insets all closed content by cornerRadiusInsets.closed.bottom (14 pt) horizontally. So the glow floated inside the notch as its own rounded rect — clipped top 6 / bottom 20 — with near-square top corners. The notch's real rounded chin stayed black & unlit. The aurora only renders while running, which is why it looked fine at rest and boxy mid-run.

Open tab: with corner-radius scaling off, the chin was just 14 pt → square on a big panel.

after the fix

The aurora now bleeds out by -cornerRadiusInsets.closed.bottom so it reaches the real notch edges, and it's clipped to the actual NotchShape chin (not a plain rounded rect). The parent NotchShape trims the bleed → the color reads as the notch itself glowing, rounded corners and all.

Open Pi tab now uses a 24 pt chin (matching the brand-palette mockup) regardless of the corner-radius-scaling setting.

The changes

Two files. Geometry only — no behavior or palette changes.

PiPeekView.swift — aurora bleeds to the notch edge & clips to the chin
// body — aurora background
- .background(aurora)
+ .background(aurora.padding(.horizontal, -cornerRadiusInsets.closed.bottom))

// aurora — clip shape
- .clipShape(UnevenRoundedRectangle(
-     topLeadingRadius: 6, bottomLeadingRadius: 20,
-     bottomTrailingRadius: 20, topTrailingRadius: 6, style: .continuous))
+ .clipShape(NotchShape(
+     topCornerRadius: cornerRadiusInsets.closed.top,
+     bottomCornerRadius: 20))
ContentView.swift — open Pi tab keeps a soft 24pt chin
// currentNotchShape — bottom radius when open
+ if vm.notchState == .open && coordinator.currentView == .pi {
+     openBottom = 24            // soft rounded panel, even with scaling off
+ } else if Defaults[.cornerRadiusScaling] {
+     openBottom = cornerRadiusInsets.opened.bottom
+ } else {
+     openBottom = cornerRadiusInsets.closed.bottom
+ }
