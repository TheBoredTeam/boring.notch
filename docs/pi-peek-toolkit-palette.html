Draw the aurora from the
toolkit's real logo palette.

Today the peek color is one averaged blob of the Composio logo, then mechanically tinted toward indigo. Averaging a multi-color logo collapses it to mud. Instead: pull the actual brand palette from the same logos.composio.dev/api/<slug> asset the app already fetches, and weave those real colors through the mesh. Every color below is extracted from the live logos.

root cause

Why it looks muddy: averaging

→

Gmail's logo → #c8b1a5 beige mud

NSImage.averageColor() sums every pixel of Gmail's red + blue + green + yellow envelope and returns one muddy beige. legibleTint() lightens it, companionColor blends 60% to indigo — but the source was already dead. The fix isn't a better blur; it's not throwing the palette away.

Before (averaged) → after (palette mesh)

Each card: the real palette pulled from the logo SVG, then the same notch rendered the old way (top) and the new way (bottom). Mono logos — Notion, GitHub — fall back to their one brand color woven with the indigo companion, so they still get depth.

how the palette is sourced

Three cases, one rule

2+ hues

Multi-color logo

Gmail, Calendar, Slack, Drive → take the top 2–4 area-weighted dominant colors and assign them to mesh control points. The aurora is the brand.

1 hue

Single-color logo

Linear → pair the brand color with a lighter + a companion shade of itself so the mesh still has internal variation, not a flat wash.

0 hues

Mono / grey logo

Notion, GitHub → too desaturated to carry an aurora alone, so keep today's behavior: brand tone + indigo companionColor. Two-hue depth preserved.

the swift change

Palette extraction replaces the single average

In PiAgentManager.adoptLogo, swap the single averageColor for a small dominant-color pass over the cached logo, publishing a toolkitPalette: [NSColor]. PiPeekView feeds that array straight into the MeshGradient control points (corners stay pinned near-black).

// PiAgentManager.adoptLogo(_:slug:)
-image.averageColor { color in
- self.toolkitAccent = Self.legibleTint(color) // one muddy hue
-}
+image.dominantColors(max: 4) { palette in // area-weighted
+ let tinted = palette.map { Self.legibleTint($0) }
+ self.toolkitPalette = tinted // [NSColor]
+ self.toolkitAccent = tinted.first // back-compat
+ // <2 saturated colors? append companionColor (mono fallback)
+}
// PiPeekView: MeshGradient(colors: meshColors(from: pi.toolkitPalette))
