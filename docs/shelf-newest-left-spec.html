Newest image,
leftmost.

The shelf is a recency rail. The most recently added image sits at the left edge; everything older flows to the right. This document is the source of truth for shelf insertion order — every decision below is settled, and the panel beside each one is live: drop, batch-drop, and re-drop to watch the invariant hold.

The invariant

Last in, far left.

One rule governs everything: when an item enters the shelf it is inserted at index 0. The view renders the array front-to-back into a horizontal row, so index 0 is the leftmost thumbnail. Drop a new image and the rail shifts right to make room on the left.

Decision 4 · Batch drops

Drop three at once.

When several files arrive in one gesture they carry no individual timestamps. We treat the batch as a sequence and insert each at the front in iteration order — so the last item in the batch ends up leftmost, and the whole batch sits to the left of everything that was already on the shelf, in consistent newest-left order.

LIVE · batch of 3
NEWEST←
drop a batch —
OLDEST→
Batch [A,B,C] → C lands leftmost, then B, then A, all left of the older items.
Decision 3 · Re-drop

Re-dropping promotes to front.

Dropping an image that is already on the shelf doesn't create a duplicate and doesn't leave it in place — it's the most recently touched item, so it is promoted to the leftmost position. Implementation: remove the stale entry by identityKey, then insert at index 0.

NEWEST←
seed, then re-drop —
OLDEST→
Re-dropping #2 pulls it from the middle to the far left — no duplicate added.
Decision 2 · Migration

Existing shelves reverse once.

Items saved by older builds are stored oldest-first (the old append order). On the first launch under the new build the persisted array is reversed exactly once — guarded by ShelfNewestLeftMigrationV1 in UserDefaults — so your existing items adopt the newest-leftmost invariant immediately instead of looking inconsistent until the shelf churns over.

Settled decisions

What we agreed in the grill.

DECISION 1

Order basis: insertion recency

Newest-added is leftmost — not a timestamp sort. No new model field; works uniformly for files, links, text, and screenshots.

insert(at: 0), no schema change
DECISION 2

Migration: reverse existing once

Persisted oldest-first arrays are reversed a single time on first launch, gated by a UserDefaults flag.

ShelfNewestLeftMigrationV1
DECISION 3

Re-drop promotes to front

A duplicate (same identityKey) is removed and re-inserted at index 0. No orphaned files — identityKey is path/content-based.

removeAll + insert(at: 0)
DECISION 4

Batch: last in batch leftmost

Each item in a multi-drop inserts at the front in iteration order, so the batch sits left of older items, last-processed leading.

per-item insert(at: 0)
Implementation

The exact change.

ShelfStateViewModel.swift · add(_:) — insertion + promote + batch

func add(_ newItems: [ShelfItem]) {
    guard !newItems.isEmpty else { return }
    var merged = items
    // Newest-leftmost: insert each at front in iteration order (last in batch = leftmost).
    // Re-dropped item (same identityKey) is promoted: remove stale entry, re-insert at 0.
    for it in newItems {
        let key = it.identityKey
        merged.removeAll { $0.identityKey == key }
        merged.insert(it, at: 0)
    }
    items = merged
}

ShelfStateViewModel.swift · init() — one-shot reverse migration

private init() {
    var loaded = ShelfPersistenceService.shared.load()
    if !UserDefaults.standard.bool(forKey: Self.newestLeftMigrationKey) {
        loaded.reverse()  // oldest-first → newest-first, exactly once
        UserDefaults.standard.set(true, forKey: Self.newestLeftMigrationKey)
    }
    items = loaded
}

ShelfView.swift · content — snap to leading on new arrival

ScrollViewReader { proxy in
    ScrollView(.horizontal) {
        HStack { ForEach(tvm.items) { item in ShelfItemView(item: item).id(item.id) } }
    }
    // New items land at index 0; snap back to leading so they're visible.
    .onChange(of: tvm.items.first?.id) { _, frontID in
        guard let frontID else { return }
        withAnimation(Motion.resolved(Motion.shelfItemEnter, reduceMotion: reduceMotion)) {
            proxy.scrollTo(frontID, anchor: .leading)
        }
    }
}
