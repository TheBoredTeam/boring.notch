01

Tool calls, streamed —
not announced after the fact.

Today the notch only learns about a tool when tool_execution_start fires — after the model has finished writing every argument. For a long email body that's seconds of generic “Thinking…” followed by a sudden jump. The pi SDK already emits toolcall_start the instant the model commits to a tool; the sidecar just drops it. We surface it: the sanitized name appears while the call is still forming, shimmering until it executes, then settling solid. Shimmer = forming. Solid = running. ✓ = done.

02

Pinned stays put.
Everything else closes.

The current rule holds the panel open when there's draft text or a running task — which means the notch sometimes refuses to close and the user can't tell why. New rule, one sentence: the panel stays open without hover if and only if you pinned it. Not pinned → unhover always closes (a running task collapses to the §01 peek, so nothing is lost). And since the panel now sizes itself to its content (§03), the old expand chevron has exactly one job left: it's a pin — and it must look pinned, so the behavior is never a mystery.

03

The panel hugs the conversation —
it grows from a base, never traps a scroll.

This is field-sizing: content applied to the whole panel — the Dynamic Island rule. The Pi panel starts at its base height (190pt). As the response streams in, the answer area expands with the text and the panel grows downward with it, line by line, up to the 360pt max — and only past that does inner scrolling engage, as the documented last resort. Under the text rides a small blank footer: dynamic empty space that moves down as the panel grows, so the newest streamed words never sit on the panel's edge. The prompt field follows the same rule (grows 1→6 lines; its growth is panel growth too). The result: an idle Pi tab is small, a working Pi tab is exactly as big as what it's saying — the notch is sized by the conversation, not by a constant.

Typing → field/panel growth is INSTANT (frequency

04

Locked decisions.

Streaming source of truth

Sidecar maps SDK message_update / toolcall_start → new wire event tool_forming. The Swift side treats it as "name known, not yet executing." No partial-args parsing in v1 — the name is enough to drive the whole UI.

Shimmer semantics

Shimmer means forming, solid means executing, ✓ means done. One visual language across the peek and the open tab's activity line. Linear sweep, 1.8s loop, killed under Reduce Motion.

Spacing

16pt between the notch cutout and the sanitized tool text (was 6pt). 14pt between the toolkit logo and the cutout, and 14pt from the pill's outer edge (was flush). Both wings breathe; the cutout reads as hardware, not as a divider the content leans on.

Hold-open has exactly one reason

Pi panel stays open without hover ⟺ user pinned it. Draft text and running tasks no longer hold the panel — the draft persists in memory, the run collapses to the peek. Share-sheet sessions (existing, app-wide) remain untouched.

The pin is visible, and it's only a pin

The old expand chevron becomes a pin (SF Symbol pin/pin.fill, accent-tinted + ring while engaged). It controls hover-independence and nothing else — geometry always belongs to the content. Defaults[.piExpanded] is replaced by Defaults[.piPinned].

The panel hugs the conversation (Dynamic Island rule)

Panel height = clamp(content height, 190pt base, 360pt max), always. The streamed response grows it on the existing height spring (continuous, interruptible); typing grows it instantly (frequency rule); past max, inner scroll engages as the last resort. A 28pt blank comfort footer rides under the text the whole way.

Prompt field grows, same rule

TextField(axis: .vertical).lineLimit(1...6) — its growth is panel growth. ⇧↵ for newline, ↵ to send.

Reduce Motion is a first-class path

Every new animation has a defined fallback: shimmer → static, wave → static bars, glow → fade, swaps → crossfade. Mirrors the existing Motion.resolved() pattern.
