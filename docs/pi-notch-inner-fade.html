Inner-fade glow — detailed

The running-agent aurora stays inside the notch silhouette, but a vertical fade dissolves the color to black before it reaches the chin — so there's no hard lit edge, just a soft glow pooled toward the top that melts into the notch. Today vs inner-fade

Same palette, same anchors, same blur. The only difference is the bottom of the glow.

Today — hard clip. Color runs all the way to the rounded chin and stops at the clip edge → a crisp lit border around the bottom. The “mechanical” read.
Inner-fade. Color dies out before the chin; the bottom of the notch is plain black, so the glow reads as a soft pool, not a panel.

How it maps to PiPeekView.swift

One added modifier on the existing aurora — a vertical mask (opaque at the top of the notch, clear toward the chin). Everything else — anchors, blooms, blur, the NotchShape clip, the swell/opacity animation — is untouched.

// inside `aurora`, after .compositingGroup().blur(...) and BEFORE the NotchShape clip:
.mask(
    LinearGradient(
        stops: [
            .init(color: .white,          location: 0.0),   // pooled at the top
            .init(color: .white.opacity(0.55), location: 0.52), // fade start (tunable)
            .init(color: .clear,          location: 1.0),   // gone before the chin
        ],
        startPoint: .top, endPoint: .bottom
    )
)
// .clipShape(NotchShape(...))  ← still rounds the (now-faded) bottom; no hard color edge left
