● Status dots — glanceable, no idle motion

Green — ACTIVE · ready to use
Yellow — INITIALIZING / INITIATED · pending your action
Red — EXPIRED/FAILED/INACTIVE/REVOKED · needs re-auth
Dots transition color in ~150ms on state change and never animate at rest — you glance at these constantly, so motion here would be noise (Emil's frequency rule). A red dot also surfaces as a static badge on the resting menu-bar icon so you know to open it, without anything pulsing in your periphery.

✓ Decisions locked

  • D1
    Default account = the extension's config file ~/.pi/agent/extensions/composio-x-pi.json → defaultAccounts. It's the only store the agent's account-resolver.ts actually reads — there is no Composio “set-default” API. The sidecar reads + writes it; the system prompt references it; the old UserDefaults + prompt-hint path is retired. UI and agent can no longer disagree.
  • D2
    One agent, two views. PiAgentManager.shared is a singleton — both surfaces observe the same @Published state. No second process, no second session.
  • D3
    Surfaces split by frequency. Notch hover-expand = aurora + branding (rich); menu-bar app = native glass connections list (quiet); collapsed peek = quiet.
  • D4
    Connect via the agent's capability, not a catalog. authorize(app) → browser → waitForConnection(), mirroring the extension's own composio_manage_connections.
  • D5
    Expiry detection on localhost = SDK WebSocket triggers.subscribe() (outbound, no public URL, no signature verification) + 5-min poll + reconcile-on-open. The HMAC webhook-signature path is Composio's production/public webhook channel — not needed for a local app, so we don't stand up a webhook server.

→ “Set default” data flow (why it's now real)

1
You tap ★ on Gmail “personal” in the menu-bar app.
2
Swift sends set_default { toolkit:"gmail", selector:"personal" } over stdin to the sidecar.
3
Sidecar writes composio-x-pi.json → defaultAccounts.gmail = "personal" (the authoritative file) and calls connectedAccounts.update(id,{alias}) to confirm the server alias.
4
Agent's resolveAccount("gmail") reads that same file → uses personal, deterministically. The system-prompt block, now sourced from the same file, agrees.
5
Menu-bar app re-reads the file on open → ★ always reflects exactly what the agent will do.
Today the UI writes UserDefaults → a system-prompt hint the resolver ignores, while the agent reads the config file — so they silently disagree. D1 collapses that into one source of truth.
