How to resolve the macOS-15 / 14 split

MeshGradient needs macOS 15. The app's floor is 14.0. One choice, your call:

A · Hero + fallback recommended

Native MeshGradient behind if #available(macOS 15, *), layered-radial for 14. You see the hero look; 14 users still get a tasteful gradient.

  • No project setting changes
  • Two code paths to keep as siblings
  • Widest compatibility

B · Bump floor to 15.0 simpler code

Set MACOSX_DEPLOYMENT_TARGET = 15.0, drop the fallback entirely. glow/edgeBloom become straight mesh gradients.

  • One clean code path
  • Cuts off macOS 14 users
  • Edits the .pbxproj (6 configs)
step 2b · the swift shape

What replaces glow + edgeBloom

Each becomes an @available-gated @ViewBuilder: a real mesh on 15, the layered radials on 14. Corners stay pinned near-black so the aurora bleeds into the notch instead of forming a hard rectangle; interior points carry accentColor / companionColor and breathe on toolCallActive.

-RadialGradient(colors: [accent.opacity(0.8),
- companion.opacity(0.4), .clear], center: .center, …)
// glow + edgeBloom → one gated aurora fill
+@ViewBuilder private var auroraFill: some View {
+ if #available(macOS 15, *) {
+ MeshGradient(width: 3, height: 3,
+ points: meshPoints, // corners pinned, interior breathes
+ colors: meshColors) // accent / companion + ~black corners
+ .blendMode(.plusLighter) // light, not paint
+ } else {
+ // layered RadialGradients, screen-blended (the 14 sibling)
+ }
+}

All existing plumbing stays: pi.isRunning, toolCallActive, reduceMotion, Motion.glowBloom, scale/opacity transitions, .allowsHitTesting(false). Under Reduce Motion the mesh points hold static — only the opacity/scale bloom remains.
